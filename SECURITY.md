# Security Policy

## Supported versions

The latest commit on the default branch is supported. Generated books and documentation releases are immutable snapshots; security or integrity fixes are made in a new revision.

## What counts as a security issue

This project is primarily static documentation, but reports are welcome for:

- malicious or unexpectedly executable content in examples or build tooling;
- dependency or workflow configuration that exposes credentials;
- unsafe command instructions that could damage a contributor's system;
- cross-site scripting or content-injection behavior in the published site;
- compromised release or artifact provenance.

Content inaccuracies and ordinary code bugs belong in the public issue tracker, not the security channel.

## Private reporting

Use [GitHub private vulnerability reporting](https://github.com/vinayreddykalluri/SDE2-Interview-Handbook/security/advisories/new). Include the affected path, impact, reproduction steps, and any proposed mitigation.

If private vulnerability reporting is unavailable, contact the repository owner through [GitHub](https://github.com/vinayreddykalluri) and request a private reporting channel. Do not disclose exploitable details in a public issue.

## Response process

The maintainer will acknowledge a valid report, assess severity, prepare a fix, and coordinate disclosure. No fixed response-time guarantee is offered, but good-faith reports will be handled as promptly as practical.
