# SDE-2 Readiness Matrix

Use evidence rather than confidence. A topic is not complete because it feels familiar; it is complete when you can produce the expected artifact under interview constraints and answer follow-up questions.

## Scoring levels

| Level | Meaning | Observable behavior |
| --- | --- | --- |
| 0 | Not yet covered | Cannot define the concept or start independently |
| 1 | Assisted | Solves with prompts; misses edge cases or trade-offs |
| 2 | Interview ready | Drives a correct answer and handles normal follow-ups |
| 3 | Strong SDE-2 | Compares alternatives, handles failure, and connects to production |

## Competency checklist

| Competency | Evidence required | Target |
| --- | --- | --- |
| Coding correctness | Tested implementation plus edge-case trace | 3 |
| Pattern selection | Trigger, invariant, and rejected alternatives | 2 |
| Complexity | Time and space derivation, including hidden costs | 3 |
| LLD | Class, sequence, state, and extension strategy | 2 |
| HLD | Requirements, estimates, APIs, data, topology, failures | 2 |
| Data systems | Schema, indexes, consistency, partitioning | 2 |
| Distributed systems | Delivery, ordering, coordination, retries | 2 |
| Reliability | SLO, timeout, backpressure, degradation | 2 |
| Security | Trust boundaries, auth, abuse, secrets, audit | 2 |
| Operations | Metrics, logs, traces, rollback, runbook | 2 |
| Leadership | Six evidence-backed stories with outcomes | 2 |

## Evidence log template

```text
Date:
Competency:
Prompt:
Artifact or recording:
Score before review:
Missed signal:
Specific corrective drill:
Score after retry:
```

Re-score only after a new timed attempt. Reading an explanation does not raise the score.
