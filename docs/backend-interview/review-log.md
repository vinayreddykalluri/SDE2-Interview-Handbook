## Monthly checkpoint

- Which concepts moved from recognition to independent application?
- Which errors repeated across different prompts?
- Which module has not been reviewed under time pressure?
- Which design lacks a failure, security, or operational deep dive?
- Which behavioral story lacks measurable evidence or ownership?
- What can leave the study queue because it is durable?

Promote a module only after a new timed attempt earns the higher score. Demote it immediately when a mock exposes a foundational gap.
