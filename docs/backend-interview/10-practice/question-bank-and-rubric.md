# Question Bank and Mock-Interview Rubric

## Prompt bank

### Programming

- Longest valid event window under changing constraints.
- Dependency ordering with cycle reporting.
- Top-K frequent items in a bounded-memory stream.
- Concurrent cache with expiry and eviction.

### LLD

- Rate limiter, job scheduler, notification platform, parking lot, or expense splitter.

### HLD

- Chat, news feed, file storage, checkout, metrics ingestion, webhooks, feature flags, or distributed scheduler.

### Production and behavioral

- Diagnose a latency regression after deployment.
- Respond to duplicate payment events.
- Explain a decision trading delivery speed against reliability.
- Describe a conflict where your preferred design was rejected.

## Scoring rubric

Score each dimension from `0-3` using the readiness levels.

| Dimension | What the interviewer should observe |
| --- | --- |
| Clarification | Users, contract, constraints, and exclusions |
| Structure | Coherent sequence and time budget |
| Correctness | Invariants and edge/failure cases |
| Depth | Internals rather than technology names |
| Trade-offs | Alternatives compared using requirements |
| Communication | Decisions narrated; feedback incorporated |
| Operations | Security, observability, rollout, recovery |
| Ownership | Decisions, risk acknowledgement, closure |

## Feedback record

Capture one strongest signal, one missed signal, one concrete drill, and the retry date. Avoid feedback such as "study system design more" because it cannot drive measurable correction.
