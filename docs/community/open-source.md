# Open Source and Licensing

The handbook is designed for public reading, reuse, teaching, translation, correction, and extension.

## Dual-license model

| Material | License | Practical meaning |
| --- | --- | --- |
| Java source, Python scripts, workflows, and configuration | MIT | use, modify, distribute, sublicense, and include commercially with the required notice |
| Handbook prose, diagrams, exercises, and revision material | CC BY 4.0 | share and adapt for any purpose with attribution and change disclosure |

The root [MIT license](https://github.com/vinayreddykalluri/SDE2-Interview-Handbook/blob/master/LICENSE) and [content-license notice](https://github.com/vinayreddykalluri/SDE2-Interview-Handbook/blob/master/LICENSE-CONTENT.md) define the applicable terms.

## Attribution

When reusing handbook content, identify the project, link to the repository, link to CC BY 4.0, and state whether you changed the material.

Suggested attribution:

> Adapted from the SDE2 Interview Handbook by Vinay Reddy Kalluri and contributors, licensed under CC BY 4.0.

## Contribution rights

Only submit material you created or have the right to contribute. External material must have a compatible license and clear attribution. Do not copy proprietary interview questions, confidential hiring material, paid-course content, or copyrighted diagrams without permission.

## Open development

Technical decisions, corrections, and roadmap discussions should happen in public issues and pull requests unless they involve private security or conduct reports. Generated artifacts are reproducible from the public source tree.
